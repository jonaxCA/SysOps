Comparación de algoritmos

Las comparaciones usan un motor headless determinista para correr decenas de simulaciones al instante. El simulador en vivo sigue usando workers reales.

Limitaciones
  • El motor de comparación es síncrono y no usa Workers (sería demasiado lento correr 9 algoritmos en paralelo real).
  • Las métricas son deterministas, pero la cadena de páginas varía con el PID (semilla = hash del PID).
  • El highlight verde marca el mínimo en una sola métrica (TAT, faults, o tiempo total según el barrido).

Limitaciones
  • Solo non-preemptive (FCFS, SJF, HRRN, Priority). Preempción real con fork requiere SIGSTOP/SIGCONT, que no es portable a Windows.
  • No hay módulo de memoria en el backend — se enfoca solo en demostrar fork().
  • Si el backend no está corriendo verás "error de conexión".
  • El consumo de CPU es real: spawning 30 forks simultáneos puede saturar tu equipo.

Cómo usar el simulador

  1. Carga procesos en la pestaña Procesos: agrégalos a mano, usa una demo, o sube un archivo.
  2. Configura en Simulación: # de cores (cada uno = thread real), algoritmo, quantum si aplica, y memoria.
  3. Pulsa ▶ Start. Verás los cores ejecutando procesos en paralelo y el Gantt con una fila por core.
  4. Mira la pestaña Memoria en cualquier momento para ver page faults y reemplazos.
  5. Usa Comparar para correr decenas de simulaciones de un click y ver cuál algoritmo gana.
  6. Para demostrar fork() real, abre el backend Node y usa la pestaña Fork Backend.

Conceptos clave

Web Worker
API del navegador que corre JS en un thread del SO separado. Aquí: 1 worker = 1 core simulado.
fork()
Llamada UNIX que crea un proceso hijo. En el backend Node, cada proceso del simulador es un fork real con su PID.
Quantum
Tiempo máximo que un proceso puede correr antes de ser desalojado (RR, MLQ, MLFQ).
Page Fault
El proceso accede a una página que no está en memoria física. El reemplazo decide a quién sacar.
Affinity
Restricción de en qué cores puede correr un proceso. Útil para escenarios de balance/desbalance.
Speedup
burst total / tiempo real. Mide la ganancia por paralelismo (ideal: igual al # de cores).
