Threads del mismo proceso comparten páginas (menos faults). Forks duplican la memoria (más faults).

Limitaciones
  • Las referencias se generan con un patrón sintético (25% salto, 30% secuencial, 45% localidad).
  • Óptimo necesita conocer el futuro — usa la cadena precomputada por proceso.
  • Fragmentación interna se aproxima asumiendo uso proporcional.
  • Memoria debe ser múltiplo de page size (se redondea hacia abajo).
  • Esta simulación es independiente de Scheduling — corre en su propio motor.

Comparación de algoritmos

Las comparaciones usan un motor headless determinista. Los simuladores en vivo (Scheduling y Paginación) siguen usando workers reales.

Limitaciones
  • El motor de comparación es síncrono (no usa workers).
  • Cadena de páginas determinista por PID (semilla = hash del PID).
  • Highlight verde: mínimo en una sola métrica.
  • La comparación usa la lista expandida (con threads y forks).

Cómo usar el simulador

  1. Procesos: agrega procesos con PID, arrival, burst, prioridad, páginas, threads (T) y forks (F).
  2. Scheduling: configura # cores y algoritmo. Pulsa Start. Verás los cores ejecutando los ejecutables expandidos en paralelo.
  3. Paginación: configura memoria/page size/algoritmo de reemplazo. Pulsa Start. Verás cómo entran y salen páginas.
  4. Comparar: ejecuta todos los algoritmos sobre los mismos procesos para ver cuál gana.
  5. Modo RIP & TEAR: estética Doom para Scheduling. La lógica es la misma, solo la presentación cambia.

Conceptos clave

Web Worker
API del navegador que corre JS en un thread del SO separado. 1 worker = 1 core simulado.
Thread (T)
Hilo de ejecución dentro de un proceso. Comparte memoria con sus hermanos. T=3 genera 3 ejecutables que comparten páginas.
Fork (F)
Copia independiente del proceso. Cada fork tiene su propia memoria. F=2 genera original + 2 forks separados.
Quantum
Tiempo máximo que un proceso puede correr antes de ser desalojado (RR, MLQ, MLFQ).
Page Fault
Acceso a una página que no está en memoria física. El reemplazo decide a quién sacar.
Affinity
Restricción de en qué cores puede correr un ejecutable.
Speedup
burst total / tiempo real. Mide la ganancia por paralelismo.
